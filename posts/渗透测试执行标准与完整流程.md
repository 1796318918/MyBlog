# 什么是渗透测试？

渗透测试就是请人模拟黑客来攻击你的系统，目的是找出薄弱点，然后在真黑客发现之前修好。和漏洞扫描不一样，扫描只是给个清单，渗透测试更强调攻击路径的组合和实际影响证明。

按信息掌握程度，有黑盒（啥也不知道）、白盒（啥都给）和灰盒（给一部分）。常见的测试对象有网络、Web 应用、移动 App，甚至包括社会工程学（比如假装客服打电话）。

# PTES 标准七个阶段

PTES 是业内比较认可的一套渗透测试流程标准，把整个过程拆成了七步。

**1. 前期交互**：开工前最重要的事，把测试范围、规则、时间、授权都谈清楚，签好书面授权。这步没做好，搞不好就变成真入侵了。

**2. 信息收集**：这是花时间最多的阶段。被动收集就是利用公开资源，比如查 WHOIS、搜 Google Hacking、在 Shodan 上查目标暴露的设备、通过 GitHub 找泄露的密码。主动收集就是直接和目标交互，比如用 Nmap 扫端口、拿 gobuster 爆破目录、枚举子域名。

**3. 威胁建模**：把收集的信息拼起来，分析哪些资产最值钱，攻击路径怎么走。比如外网有个 Web 应用，可能通过文件上传拿到 Shell，再横向移动进内网。

**4. 漏洞分析**：结合扫描器和手工测试，确认哪些漏洞真的能用。有时候扫描器报的高危可能实际上利用不了，就需要自己动手试试。

**5. 渗透攻击**：开始“打”了。利用 SQL 注入拖库，用文件上传漏洞传 WebShell，或者用 Metasploit 针对某个未打补丁的服务发起攻击。

**6. 后渗透**：攻进去之后，就要维持权限（防止重启后掉了），在内网里到处看，提权、偷数据、横向移动，直到拿到最终目标。

**7. 报告**：最后把过程写清楚，管理层看业务影响，技术团队看复现步骤和修复建议。好的报告才能推动问题解决。

# 常用工具

- 信息收集：Nmap, theHarvester, subfinder
- 漏洞利用：Metasploit, sqlmap
- 密码攻击：Hydra, Hashcat
- 后渗透：Mimikatz, BloodHound

记住，渗透测试一定要在法律和授权框架内进行，否则就是犯罪。白帽子的价值在于帮助修复，而不是破坏。
