# 为啥要自动扫描？

手工测试固然精准，但企业里动辄几百上千台机器，一个个手测根本不现实。漏洞扫描器就是帮我们快速发现已知漏洞的工具，它内置了海量的漏洞特征库，对着目标扫一遍，就能列出一堆潜在问题。

比较有名的商业扫描器有 Nessus、Qualys，开源的有 OpenVAS。对于我们学习来说，Nessus 有免费的家庭版（限制 IP 数），OpenVAS 完全免费，都很合适。

# Nessus 上手

Nessus 的安装非常简单，去官网下载对应系统的安装包，执行一下就好。装好之后启动服务，浏览器访问 `https://localhost:8834` 就能看到管理界面了。第一次进去会初始化，花几分钟更新插件。

扫之前先创建一个扫描任务。选模板的时候，一般网络扫描用 “Basic Network Scan”，Web 应用就用 “Web Application Tests”。填上目标 IP 或域名，如果需要深度扫描，最好能提供系统的账号密码（授权情况下），这样能拿到更多信息。

点开始扫描后，根据目标数量，可能要跑十几分钟到几小时。扫完以后，报告会按严重程度（Critical、High、Medium、Low）给你列出来，每条漏洞都有详细说明、CVSS 评分和修复建议，非常贴心。你可以导出 PDF 或 HTML 报告发给领导看。

# OpenVAS 怎么玩？

OpenVAS 是 Greenbone 公司维护的开源方案，功能和 Nessus 差不多，就是界面稍微原始一点。推荐用 Docker 装，一条命令拉镜像跑起来就完事。

管理界面叫 GSA，访问 `https://localhost:9392`。第一次更新漏洞库要很久，可能得几个小时，耐心等待。之后创建任务、扫、看报告，流程和 Nessus 差不多。

# 扫描的注意事项

- 一定要有授权！没授权去扫别人是违法的。
- 扫描会产生流量，可能把老旧设备扫挂掉，最好选业务低峰期。
- 扫描器会有误报，报告里说的高危漏洞不一定真的能用，需要人工验证一下。
- 别扫一次就完事了，安全是个动态过程，定期扫描才能发现新问题。

漏洞扫描是安全的第一步，帮我们快速定位短板，但后面还得结合手工测试才能真正评估风险。
