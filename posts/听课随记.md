# Image Classification的挑战

- 角度不同
- 光照不同
- 背景杂乱
- 缩放
- 遮挡
- 每个类别内部的差异（如男女、猫的不同颜色品种）

都会造成图像的每一个像素都发生变化，这对计算机来说就是一个完全新的图片。

# 最近邻分类器 Nearest Neighbor Classifier

一句话说：

​ **训练过程把数据和标签都存入内存。**

​ **预测过程使用预测图像和内存中所有图像比对，计算距离，输出距离最小的样本标签。**

```python
class NearestNeighbor:
    def __init__(self):
        pass

    def train(self, X, Y):
        """把训练数据和标签都存入内存"""
        self.Xtr = X
        self.ytr = Y

    def predict(self, X):
        num_test = X.shape[0]
        Ypred = np.zero(num_test, dtype = self.ytr.dtype)  # 测试样本长度，类型和输入标签一致

        for i in range(num_test):
            distances = np.sum(np.abs(self.Xtr - X[i,:]), axis = 1)  # 计算 L1 距离
            min_index = np.argmin(distances)  # 取最小距离索引
            Ypred[i] = self.ytr[min_index]

        return Ypred
```

问题：

​ 训练简单，但是工作全都堆到了预测阶段，**预测时间长**。

## 距离函数

- L1距离，曼哈顿（Manhattan）距离

  $$
  L_1 = \sum_p \| I_1^p - I_2^p \|
  $$

  ![image-20260629152428572](/images/听课随记/image-20260629152428572.png)

-

- L2距离，欧氏距离（Euclidean）距离

  $$
  L_2 = \sqrt{ \sum_p{( I_1^p - I_2^p )^2} }
  $$

  ![image-20260629152512069](/images/听课随记/image-20260629152512069.png)

  所以$L_2$对特征轴怎么取都是稳定的，但$L_1$​就很依赖特征轴到底是怎么取的。

## Softmax Classifier

使用线性分类器出来的结果没有一个具体的上下界限，可能有的很大有的很小，有的是正数有的是负数，这就不好判断到底属于什么类别。

例如

![image-20260629161236475](/images/听课随记/image-20260629161236475.png)

Softmax的方法就就是把线性分类器得到的结果转换为各类别的概率，目标函数就是取负取对数（让概率最大化问题转化为最小化问题即目标函数）：

$$
P(Y=k | X=x_i) = \frac{e^{s_k}}{\sum_j {e^{s_j}}}\\
L_i = -log P(Y=y_i | X=x_i) = -log(\frac{e^{s_k}}{\sum_j {e^{s_j}}})
$$
